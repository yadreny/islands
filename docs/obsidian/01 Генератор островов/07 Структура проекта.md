# Структура проекта

## Актуальная схема

```text
D:\codex\islands
- docs/
  - obsidian/
- specs/
  - island-generator/
- tools/
- islands/        <- Unity-проект
```

## Правило

Unity-проект живёт не в `game/unity`, а в `D:\codex\islands\islands`.

Это надо считать основной рабочей директорией проекта Unity для всех дальнейших задач по коду, сценам, ассетам и структуре `Assets`.

## Назначение папок

- `docs/obsidian` — документация, техдизайн, заметки и лор.
- `specs/island-generator` — формальные контракты, схемы и таблицы параметров.
- `tools` — вспомогательные скрипты и валидация.
- `islands` — Unity-проект.

## Практическое следствие

Если задача касается Unity-кода или структуры Unity, рабочим корнем считается `D:\codex\islands\islands`.
