# Генерация рек

## Цель

На этом этапе генератор строит не финальную гидрологию и не озёра, а речной гидроскелет острова: набор основных русел, которые уже можно использовать для дальнейшего рельефа, городов и дорог.

## Текущий контракт

```text
IslandRiverGenerator
- input: IslandShapeRequest
- output: Vector4[][]
```

## Семантика выхода

Каждая река — это массив `Vector4`.

- `x` — локальная координата точки русла по оси `X`.
- `y` — вертикальная координата. Пока для gizmo-слоя не используется и остаётся `0`.
- `z` — локальная координата точки русла по оси `Z`.
- `w` — ширина/полноводность в этой точке русла.

## Что регулирует количество рек

Количество рек сейчас задаётся в пресете острова через `riverAbundance`.

```text
IslandPreset
- riverAbundance   // 0..1
```

Смысл:

- `0` — остров практически без выраженной речной сети.
- `1` — остров с насыщенной речной сетью.

Итоговое число рек дополнительно зависит от:

- площади острова;
- сложности рельефа;
- конкретного сидa.

## Как сейчас строятся реки

Пока это именно гидроскелет от формы острова, а не строгая симуляция стока.

Пайплайн сейчас такой:

1. Генерируется контур острова.
2. По контуру выбираются устья рек.
3. Для каждого устья строится внутренняя точка-исток.
4. Между истоком и устьем строится плавное русло.
5. Для каждой точки русла записывается `Vector4`, где `w` растёт к устью.

## Что важно

- Это временный речной слой до нормальной генерации рельефа.
- Реки сейчас строятся от формы острова и внутренней логики пресета.
- Озёра сюда пока не входят и будут отдельным этапом.
- Когда появится рельеф, речной слой лучше перевести из «shape-first hydroskeleton» в более честную симуляцию стока.

## Gizmo preview

Для просмотра используется отдельный `MonoBehaviour`:

```text
IslandRiverPreview
```

Он:

- запускает `IslandRiverGenerator`;
- рисует основной контур острова;
- рисует русла рек gizmo-линиями;
- может показывать точки русла с радиусом из `w`.
