# Генерация формы острова

## Цель

На этом шаге генератор должен выдать не рельеф и не биомы, а убедительный контур острова с понятной крупной логикой. Выход этого этапа должен быть достаточно стабильным, чтобы на него уже можно было сажать контроль, города и древний слой.

## Вход

Используется внешний контракт из [[01 Генератор островов/02 Форма острова и shape API]]:

```text
IslandShapeRequest
- preset
- targetArea
- targetMaxElevation
- aspectPercent
- reliefComplexityPercent
- direction
- seed
```

И скрытые поля пресета:

```text
IslandPreset
- recommendedArea
- recommendedMaxElevation
- recommendedAspectRatio
- recommendedReliefComplexity
- footprintFill
- massLayoutType
- reliefProfile
```

## Что должно выйти

```text
IslandShapeResult
- coastline
- area
- boundingFrame
- majorAxisLength
- minorAxisLength
- orientation
- mainComponentMask
- offshoreIslets
- macroFeatures
```

### Что здесь важно

- `coastline` — финальный контур главного острова.
- `boundingFrame` — рабочий габарит формы до посадки рельефа и гидроскелета.
- `offshoreIslets` — спутниковые островки, если пресет их допускает.
- `macroFeatures` — перечень крупных жестов формы: бухты, полуострова, шеи, лагуны, обрывы кольца и так далее.

## Производные величины

Сначала из входа вычисляются рабочие параметры формы.

```text
finalArea = targetArea
finalMaxElevation = targetMaxElevation
finalAspectRatio = 1 + (recommendedAspectRatio - 1) * aspectPercent / 100
finalReliefComplexity = recommendedReliefComplexity * reliefComplexityPercent / 100
```

Чтобы из площади и вытянутости получить рабочий габарит, нужен `footprintFill`.

```text
majorAxisLength = sqrt(finalArea * finalAspectRatio / footprintFill)
minorAxisLength = sqrt(finalArea / (finalAspectRatio * footprintFill))
```

Это даёт честную рамку, внутри которой потом строится силуэт.

## Внутреннее представление

Лучше строить форму не сразу как полигон, а как плотностное поле суши.

```text
landField(x, y)
< 0  => суша
= 0  => береговая линия
> 0  => море
```

### Почему это удобно

- проще собирать форму из нескольких масс;
- проще вырезать бухты и лагуны;
- проще контролировать кольца, перешейки и спутниковые островки;
- проще сглаживать и чистить контур перед извлечением полигона.

Полигон берега извлекается только в самом конце.

## Нормализованный рабочий домен

Форму удобно собирать в локальной системе координат:

- центр острова в `(0, 0)`;
- главная ось совпадает с осью `X`;
- вторичная ось совпадает с осью `Y`;
- поворот `direction` применяется уже к готовой форме.

Это снимает лишнюю сложность с примитивов формы.

## Этапы алгоритма

### 1. Resolve Preset

Из пресета берутся:

- тип крупной массы;
- тип вертикального профиля;
- базовая вытянутость;
- базовая сложность;
- плотность заполнения габарита.

### 2. Build Base Mass

Строится самая грубая масса острова. На этом этапе ещё не должно быть мелкой береговой линии.

#### Варианты по `massLayoutType`

- `SingleCore` — один главный массив, обычно эллипс или слегка лопастная масса.
- `DoubleCore` — два главных центра суши с контролируемой шеей между ними.
- `Arc` — вытянутая масса с мягким изгибом.
- `ContinentalBlock` — большой старый блок с 2-3 крупными выступами.
- `BrokenBlock` — старый блок с заранее заложенной асимметрией и разломанностью.
- `Ring` — кольцо с внутренней лагуной.
- `DrownedRelief` — масса с заранее заложенными крупными врезами и почти внутренними морями.

### 3. Apply Macro Features

На базовую массу накладываются крупные жесты формы. Именно они делают остров узнаваемым.

Основные операции:

- `AddLobe` — добавить крупную долю суши.
- `CutBay` — вырезать крупную бухту.
- `ExtendPeninsula` — вытянуть полуостров.
- `CreateNeck` — сузить перешеек между двумя массами.
- `PunchLagoon` — сформировать лагуну или внутреннюю воду.
- `BreakRing` — разорвать кольцо атолла или рифовой дуги в 1-3 местах.
- `AddSatelliteIslet` — добавить островок-спутник рядом с главным массивом.
- `CarveInnerSea` — сделать крупный внутренний врез воды для drowned-типов.

### 4. Apply Directional Bias

После крупных жестов вводится асимметрия сторон:

- одна сторона может быть ровнее;
- другая — более бухтовой;
- на ветреной стороне может быть меньше удобных бухт;
- у старых континентальных островов можно усиливать только отдельные сектора, а не весь периметр.

Главное правило: изрезанность распределяется неравномерно.

### 5. Coastal Breakup

Только теперь добавляется береговая сложность меньшего масштаба. Это не чистый шум, а controlled breakup.

Параметры этого шага:

- масштаб прибрежной детализации;
- доля берега, где детализация вообще разрешена;
- тип детализации: бухтовая, мысистая, рифовая, ломано-скальная.

### 6. Cleanup And Normalize

После деформаций форму надо привести в рабочее состояние:

- восстановить целевую площадь;
- отрезать случайные шипы;
- удалить слишком мелкие артефакты;
- проверить ширину шеек;
- проверить связность главного массива;
- при необходимости смягчить слишком равномерную береговую лохматость.

### 7. Extract Coastline

Только на этом шаге из поля извлекается финальный контур берега и список спутниковых островов.

## Примитивы формы

Если формализовать конструктор, достаточно небольшого набора примитивов.

```text
ShapePrimitive
- EllipseMass
- BlobMass
- ArcMass
- RingMass
- BayCut
- PeninsulaPull
- NeckConstraint
- LagoonCut
- IsletMass
```

И набора модификаторов:

```text
ShapeModifier
- ScaleAlongMajorAxis
- ScaleAlongMinorAxis
- Bend
- Shear
- Taper
- Smooth
- ErodeSector
- ProtectSector
```

Этого уже достаточно, чтобы собрать почти все оговоренные пресеты без отдельного алгоритма на каждый.

## Инварианты качества

После генерации форма обязана пройти проверки.

### Общие

- отклонение по площади не больше допустимого порога;
- главная масса остаётся читаемой;
- нет случайных игольчатых выступов;
- нет шеек уже заданного минимума;
- форма не теряет архетип пресета.

### По типам

- `SingleCore` не должен распадаться на два равных ядра.
- `DoubleCore` должен сохранять читаемую двухдольность.
- `Arc` не должен выпрямляться в обычный эллипс.
- `Ring` должен сохранять лагуну и хотя бы частичную замкнутость кольца.
- `DrownedRelief` не должен становиться просто шумным blob без крупных врезов.

## Псевдопайплайн

```text
GenerateIslandShape(request):
  preset = ResolvePreset(request.preset)
  params = DeriveShapeParams(request, preset)
  field = BuildBaseMass(params, preset.massLayoutType)
  field = ApplyMacroFeatures(field, params, preset)
  field = ApplyDirectionalBias(field, params, preset)
  field = ApplyCoastalBreakup(field, params, preset)
  field = CleanupShape(field, params, preset)
  result = ExtractCoastline(field, params)
  ValidateShape(result, params, preset)
  return result
```

## Где заканчивается этот этап

Генерация формы острова заканчивается в момент, когда уже есть:

- большой силуэт;
- понятные сектора побережья;
- перешейки и крупные бухты;
- опорные внутренние массы суши;
- список допустимых мест для дальнейшего размещения контроля, городов и воды.

На этом этапе ещё не должно быть:

- финального рельефа;
- финальных рек;
- дорог;
- политических границ.

## Практическое правило

Если остров после генерации можно описать одной фразой вроде «старый расчленённый континентальный с крупной северной бухтой и узким южным полуостровом», значит этап формы сработал. Если его можно описать только словом «клякса», этап провален.
